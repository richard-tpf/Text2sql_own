# 知识库版本说明

## 概述

demo01 是基于 Vanna Agents 框架开发的知识库驱动智能助手。与原版 xsrtest 相比，去除了历史记忆搜索和 SQL 执行功能，新增了知识库管理能力，支持存储建表语句和业务知识，并在工作流中先搜索知识库再生成回答。

## 知识库数据结构

知识库存储在 Milvus 集合 `vanna_knowledge_base` 中：

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | VARCHAR(100) | 主键，UUID |
| embedding | FLOAT_VECTOR(1024) | 文本向量（BAAI/bge-m3） |
| content | VARCHAR(30000) | 知识内容 |
| knowledge_type | VARCHAR(50) | 类型：ddl / business |
| title | VARCHAR(500) | 知识标题 |
| timestamp | VARCHAR(50) | 创建时间 |

## 知识类型

| 类型 | knowledge_type 值 | 说明 |
|------|-------------------|------|
| 建表语句 | `ddl` | CREATE TABLE 等 DDL 语句 |
| 业务知识 | `business` | 业务规则、术语定义、流程说明等 |

## 工作流程

### 保存知识流程

```
用户提供建表语句或业务知识
    │
    ▼
Agent 识别内容类型（ddl / business）
    │
    ▼
Agent 调用 save_knowledge 工具
    │  参数：content, knowledge_type, title
    ▼
MilvusKnowledgeBase.save_knowledge()
    │  1. 生成 UUID 作为知识 ID
    │  2. 使用 BAAI/bge-m3 将内容转为 1024 维向量
    │  3. 插入 Milvus 集合并 flush
    ▼
返回保存成功确认
```

### 问答流程

```
用户提出问题
    │
    ▼
Agent 调用 search_knowledge 搜索知识库
    │  参数：query=用户问题, limit=5
    ▼
MilvusKnowledgeBase.search_knowledge()
    │  1. 使用 BAAI/bge-m3 将问题转为 1024 维向量
    │  2. 在 Milvus 中进行向量相似度搜索
    │  3. 过滤相似度 >= 0.5 的结果
    │  4. 返回按相似度排序的知识条目
    ▼
Agent 结合搜索到的知识生成回答
    │
    ▼
返回回答给用户
```

## 启动方式

```bash
python -m demo01.agent
```

访问 http://localhost:8000

## 使用示例

### 保存建表语句

```
用户：请保存以下建表语句：
CREATE TABLE t_order (
    id BIGINT PRIMARY KEY,
    user_id BIGINT NOT NULL COMMENT '用户ID',
    amount DECIMAL(10,2) COMMENT '订单金额',
    status VARCHAR(20) COMMENT '订单状态',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
) COMMENT='订单表';

Agent 行为：
  → 调用 save_knowledge(
      content="CREATE TABLE t_order ...",
      knowledge_type="ddl",
      title="t_order - 订单表"
    )
  → 返回：已成功保存建表语句到知识库
```

### 保存业务知识

```
用户：请保存以下业务知识：
订单状态说明：
- PENDING: 待支付
- PAID: 已支付
- SHIPPED: 已发货
- COMPLETED: 已完成
- CANCELLED: 已取消

Agent 行为：
  → 调用 save_knowledge(
      content="订单状态说明：...",
      knowledge_type="business",
      title="订单状态枚举说明"
    )
  → 返回：已成功保存业务知识到知识库
```

### 查询知识

```
用户：订单表有哪些字段？

Agent 行为：
  → 调用 search_knowledge(query="订单表有哪些字段？")
  → 搜索到建表语句 t_order
  → 回答：订单表(t_order)包含以下字段：
    - id: BIGINT, 主键
    - user_id: BIGINT, 用户ID
    - amount: DECIMAL(10,2), 订单金额
    - status: VARCHAR(20), 订单状态
    - created_at: DATETIME, 创建时间
```

```
用户：订单状态 SHIPPED 是什么意思？

Agent 行为：
  → 调用 search_knowledge(query="订单状态 SHIPPED")
  → 搜索到业务知识"订单状态枚举说明"
  → 回答：SHIPPED 表示"已发货"状态
```

## 配置说明

### Milvus 连接

在 `agent.py` 中配置：

```python
knowledge_base = MilvusKnowledgeBase(
    host="172.16.11.57",      # Milvus 服务地址
    port=19530,                # Milvus 服务端口
    collection_name="vanna_knowledge_base",  # 集合名称
    dimension=1024             # 向量维度（与嵌入模型匹配）
)
```

### LLM 配置

```python
llm = OpenAILlmService(
    model="deepseek-ai/DeepSeek-R1",
    base_url="https://api.siliconflow.cn/v1",
    api_key="你的API Key"
)
```

### 搜索参数调优

在 `SearchKnowledgeTool` 中可调整：
- `limit`：最大返回结果数量（默认 5）
- `similarity_threshold`：最低相似度阈值（默认 0.5，范围 0.0-1.0）
- `knowledge_type`：按类型过滤（"ddl" / "business" / 不填搜全部）
