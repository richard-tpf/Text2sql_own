# 运行流程详解

## 一、启动流程

当执行 `python -m demo02.agent` 时，程序按以下顺序初始化：

```
1. 创建 OpenAILlmService 实例
   → 连接 SiliconFlow API（https://api.siliconflow.cn/v1）
   → 加载 DeepSeek-R1 模型

2. 创建 DualKnowledgeBaseManager 实例
   → 配置 Milvus 连接参数（172.16.11.57:19530）
   → 表结构知识库集合：dw_schema_kb
   → 业务文档知识库集合：dw_business_kb
   → 实际连接在首次使用时懒加载

3. 创建辅助组件
   → SimpleUserResolver（用户认证）
   → DWSystemPromptBuilder（数仓开发专用系统提示构建）
   → NoOpAgentMemory（空的记忆实现）
   → LoggingLifecycleHook（生命周期日志）
   → LoggingLlmMiddleware（LLM 日志中间件）

4. 注册工具到 ToolRegistry
   → SaveKnowledgeTool（保存知识，支持双库路由）
   → SearchKnowledgeTool（搜索知识，支持跨库和定向搜索）

5. 组装 Agent 实例
   → 将以上所有组件注入 Agent

6. 启动 FastAPI 服务
   → VannaFastAPIServer 创建 FastAPI 应用
   → 替换默认主页为自定义主页（含双知识库入口）
   → 注册知识上传 API（单条 + 批量）
   → 注册模板下载 API
   → 注册表结构上传页面和业务文档上传页面
   → Uvicorn 监听 0.0.0.0:8000
```

## 三、数仓开发核心流程（两阶段）

根据 DWSystemPromptBuilder 中定义的开发流程，Agent 按两个阶段工作：

### 第一阶段：分析需求文档，生成开发文档

```
用户提供需求文档
    │
    ▼
┌─────────────────────────────────────────────────────┐
│ 步骤 1：解析需求文档中的指标定义表                       │
│ 逐行提取：指标名称、字段性质、统计方式、                  │
│          取值来源、取值规则、展示格式、备注               │
└──────────────────────┬──────────────────────────────┘
                       ▼
┌─────────────────────────────────────────────────────┐
│ 步骤 2：搜索表结构知识库，验证取值来源                    │
│                                                      │
│ 调用 search_knowledge(search_scope='schema')          │
│   → layer_filter='DWD' 搜索 DWD 层表结构              │
│   → layer_filter='MID' 搜索 MID 层表结构              │
│   → 确认来源表是否存在                                 │
│   → 核对字段名、类型是否匹配                            │
│   → 验证多表关联条件                                   │
│                                                      │
│ ⚠️ 未找到的表/字段 → 标注到「待确认项」，禁止虚构         │
└──────────────────────┬──────────────────────────────┘
                       ▼
┌─────────────────────────────────────────────────────┐
│ 步骤 3：搜索业务文档知识库，获取指标定义                  │
│                                                      │
│ 调用 search_knowledge(search_scope='business')        │
│   → 获取业务规则、指标口径等补充信息                     │
└──────────────────────┬──────────────────────────────┘
                       ▼
┌─────────────────────────────────────────────────────┐
│ 步骤 4：生成开发文档                                    │
│   → DWS 层表结构设计（表名、字段、DDL）                  │
│   → ADS 层表结构设计（日/周/月表）                       │
│   → 指标口径明细表                                     │
│   → 待确认项清单                                       │
└──────────────────────┬──────────────────────────────┘
                       ▼
              交给用户确认后进入第二阶段
```

### 第二阶段：根据开发文档生成加工 SQL

```
用户确认开发文档
    │
    ▼
┌─────────────────────────────────────────────────────┐
│ 步骤 1：开发 DWS 层 SQL                                │
│                                                      │
│ 调用 search_knowledge(search_scope='schema',          │
│      layer_filter='DWD') 查阅 DWD 层表结构             │
│ 调用 search_knowledge(search_scope='schema',          │
│      layer_filter='MID') 查阅 MID 层表结构             │
│                                                      │
│ → 识别加工场景（多CTE打标 / 单CTE flag / 直接聚合）      │
│ → 生成 INSERT OVERWRITE TABLE ... 的 DWS 加工 SQL      │
│ → 遵循 Hive SQL 编写规范                               │
└──────────────────────┬──────────────────────────────┘
                       ▼
┌─────────────────────────────────────────────────────┐
│ 步骤 2：开发 ADS 层 SQL                                │
│                                                      │
│ 调用 search_knowledge(search_scope='schema',          │
│      layer_filter='DWS') 查阅 DWS 层表结构             │
│                                                      │
│ → 区分时间粒度（日/周/月）                              │
│ → 派生指标（比率类）在 ADS 层计算                        │
│ → 不可加指标（COUNT DISTINCT）周/月表回源 DWD            │
│ → 生成 ADS 加工 SQL                                    │
└──────────────────────┬──────────────────────────────┘
                       ▼
┌─────────────────────────────────────────────────────┐
│ 步骤 3：执行检查清单                                    │
│                                                      │
│ → schema 前缀、LEFT JOIN 空值处理、分区条件              │
│ → 除零保护、字段名一致性、指标口径偏差                    │
│ → 关联条件、CTE 打标逻辑、派生指标位置                   │
│ → 周/月表聚合方式                                      │
└─────────────────────────────────────────────────────┘
```

## 四、知识上传流程（Web 页面）

### 表结构上传（/knowledge/schema）

支持两种方式：

```
方式一：手动输入
    用户填写表名、层级、DDL
         │
         ▼
    POST /api/knowledge/schema/upload
         │
         ▼
    DualKnowledgeBaseManager.save_schema()
         │  1. 从 DDL 提取字段名和中文 COMMENT，构建中英文混合文本
         │  2. 使用 BAAI/bge-m3 将混合文本转为 1024 维向量
         │  3. 存入 Milvus dw_schema_kb 集合
         ▼
    返回保存成功确认

方式二：MD 文件批量导入
    用户上传 .md 文件
         │
         ▼
    前端 JS 解析 MD 文件
    按 ## 库名.表名 拆分，提取层级、DDL、来源表
         │
         ▼
    逐条 POST /api/knowledge/schema/upload
         │
         ▼
    批量保存到 Milvus
```

### 业务文档上传（/knowledge/business）

```
用户选择类型（业务知识/需求文档/开发规范）
填写标题和内容
         │
         ▼
    POST /api/knowledge/business/upload
         │
         ▼
    DualKnowledgeBaseManager.save_business()
         │  1. 使用 BAAI/bge-m3 将内容转为 1024 维向量
         │  2. 存入 Milvus dw_business_kb 集合
         ▼
    返回保存成功确认
```

## 五、双知识库搜索工作原理

```
用户提问或 Agent 需要查阅知识
         │
         ▼
    search_knowledge 工具被调用
         │
    ┌────┴────────────────────┐
    │ search_scope 参数决定范围 │
    ├─────────┬───────┬───────┤
    │ schema  │  all  │business│
    └────┬────┴───┬───┴───┬───┘
         │       │       │
         ▼       │       ▼
  表结构知识库    │   业务文档知识库
  dw_schema_kb   │   dw_business_kb
  ┌────────────┐ │   ┌────────────┐
  │向量相似度搜索│ │   │向量相似度搜索│
  │可按 layer   │ │   │可按 type   │
  │过滤层级     │ │   │过滤类型     │
  │支持中英文   │ │   │            │
  │双语搜索     │ │   │            │
  └─────┬──────┘ │   └─────┬──────┘
        │        │         │
        └────────┼─────────┘
                 ▼
         合并搜索结果返回给 LLM
```

表结构知识库的中英文双语搜索原理：
1. 保存时从 DDL 中提取字段名和中文 COMMENT，构建 `表名 字段名:中文注释` 格式的混合文本
2. 用 BAAI/bge-m3 对混合文本生成向量
3. 搜索时无论用中文（如"申请金额"）还是英文（如"apply_amount"）都能命中
