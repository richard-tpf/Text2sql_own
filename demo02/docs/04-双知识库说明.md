# 双知识库说明

## 概述

demo02 采用双知识库架构，将数仓开发所需的知识分为两个独立的 Milvus 集合管理：
- 表结构知识库（dw_schema_kb）：存储建表语句、所属层级、来源表关系
- 业务文档知识库（dw_business_kb）：存储业务知识、需求文档、开发规范

两个知识库共用同一个 Milvus 连接（default alias），但使用不同的集合，数据完全隔离。

## 与 demo01 的对比

| 特性 | demo01 | demo02 |
|------|--------|--------|
| 知识库数量 | 1 个（vanna_knowledge_base） | 2 个（dw_schema_kb + dw_business_kb） |
| 知识类型 | ddl / business | 表结构：table_name+ddl+layer+source_tables；业务文档：business/requirement/standard |
| 搜索过滤 | 按 knowledge_type 过滤 | 表结构：按 layer 和 table_name 过滤；业务文档：按 knowledge_type 过滤 |
| 搜索范围 | 单库搜索 | 支持 schema / business / all 三种范围 |
| 向量生成 | 直接对内容生成向量 | 表结构：中英文混合文本向量；业务文档：直接对内容生成向量 |
| Web 管理 | 无 | 表结构上传页 + 业务文档上传页 |
| 批量导入 | 无 | 支持 MD 文件批量导入表结构 |
| 模板下载 | 无 | 需求文档模板 + 开发文档模板 + 表结构导入模板 |

## 表结构知识库

### 数据结构

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | VARCHAR(100) | 主键，UUID |
| embedding | FLOAT_VECTOR(1024) | 中英文混合文本向量 |
| table_name | VARCHAR(500) | 库名.表名，如 zijie.dwd_credit_apply_df |
| layer | VARCHAR(20) | 所属层级：DWD / MID / DWS / ADS |
| ddl | VARCHAR(30000) | 建表语句原文 |
| source_tables | VARCHAR(5000) | 来源表列表，逗号分隔 |
| timestamp | VARCHAR(50) | 创建时间 |

### 中英文双语搜索

表结构知识库的核心特性是支持中英文双语搜索。实现原理：

```
保存时：
  DDL 原文：
    CREATE TABLE zijie.dwd_credit_apply_df (
        apply_id STRING COMMENT '申请ID',
        user_id STRING COMMENT '用户ID',
        apply_amount DECIMAL(10,2) COMMENT '申请金额'
    ) COMMENT '授信申请明细表'

  → _build_bilingual_text() 提取：
    "zijie.dwd_credit_apply_df 授信申请明细表 apply_id:申请ID user_id:用户ID apply_amount:申请金额"

  → 对混合文本生成向量并存储

搜索时：
  中文搜索 "申请金额" → 命中 "apply_amount:申请金额"
  英文搜索 "apply_amount" → 命中 "apply_amount:申请金额"
  表名搜索 "credit_apply" → 命中 "zijie.dwd_credit_apply_df"
```

### 层级过滤

搜索时可通过 `layer_filter` 参数精确过滤层级，这在数仓开发中至关重要：
- 开发 DWS 层时：`layer_filter='DWD'` 和 `layer_filter='MID'`，只查看上游表
- 开发 ADS 层时：`layer_filter='DWS'`，只查看 DWS 层表
- 避免跨层误读，确保分层架构的依赖规则

### 上传方式

1. 手动输入：通过 Web 页面填写表名、层级、DDL
2. MD 文件批量导入：按模板格式编写 MD 文件，通过 Web 页面上传
3. API 调用：POST `/api/knowledge/schema/upload` 或 `/upload_batch`
4. Agent 对话：Agent 调用 `save_knowledge(save_target='schema', ...)`

## 业务文档知识库

### 数据结构

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | VARCHAR(100) | 主键，UUID |
| embedding | FLOAT_VECTOR(1024) | 文本向量 |
| content | VARCHAR(30000) | 知识内容 |
| knowledge_type | VARCHAR(50) | 类型：business / requirement / standard |
| title | VARCHAR(500) | 知识标题 |
| timestamp | VARCHAR(50) | 创建时间 |

### 知识类型

| 类型 | knowledge_type | 说明 | 示例 |
|------|----------------|------|------|
| 业务知识 | business | 业务规则、术语定义、指标口径 | 授信通过率 = 授信通过数 / 授信申请数 |
| 需求文档 | requirement | 需求方案、指标定义文档 | 授信全流程漏斗分析需求文档 |
| 开发规范 | standard | SQL 编写规范、命名规范 | 所有表名必须带 zijie. schema 前缀 |

### 上传方式

1. Web 页面：通过业务文档上传页面选择类型、填写内容
2. API 调用：POST `/api/knowledge/business/upload` 或 `/upload_batch`
3. Agent 对话：Agent 调用 `save_knowledge(save_target='business', ...)`

## 搜索工作流

### 标准开发工作流

开发数仓加工 SQL 时，Agent 按以下顺序搜索：

```
1. 搜索表结构知识库（search_scope='schema'）
   → 获取上游表 DDL 和关联关系
   → 使用 layer_filter 按层级精确过滤
   → 建议同时用中文指标名和英文字段名分别搜索

2. 搜索业务文档知识库（search_scope='business'）
   → 获取指标定义和业务规则

3. 结合两个知识库的结果，按分层规范生成 SQL
```

也可以用 `search_scope='all'` 同时搜索两个知识库。

### 搜索参数

| 参数 | 适用范围 | 说明 |
|------|---------|------|
| query | 全部 | 搜索查询文本 |
| search_scope | 全部 | all / schema / business |
| limit | 全部 | 最大返回数量，默认 5 |
| layer_filter | schema | 按层级过滤：DWD/MID/DWS/ADS |
| table_name_filter | schema | 按表名精确过滤 |
| knowledge_type | business | 按类型过滤：business/requirement/standard |

### 相似度阈值

两个知识库的默认相似度阈值均为 0.5（范围 0.0-1.0），低于阈值的结果会被过滤。

## 使用示例

### 上传表结构（通过对话）

```
用户：请保存以下建表语句到表结构知识库：
CREATE TABLE zijie.dwd_credit_apply_df (
    apply_id STRING COMMENT '申请ID',
    user_id STRING COMMENT '用户ID',
    apply_amount DECIMAL(10,2) COMMENT '申请金额',
    apply_time STRING COMMENT '申请时间'
) COMMENT '授信申请明细表'
PARTITIONED BY (ds STRING COMMENT '日期分区')
STORED AS ORC;

层级：DWD

Agent 行为：
  → 调用 save_knowledge(
      save_target="schema",
      table_name="zijie.dwd_credit_apply_df",
      layer="DWD",
      ddl="CREATE TABLE ...",
    )
  → 返回：已保存表结构 zijie.dwd_credit_apply_df 到表结构知识库
```

### 上传需求文档（通过对话）

```
用户：请保存以下需求文档：
【授信全流程漏斗分析】
| 指标名称 | 字段性质 | 统计方式 | 取值来源 | 取值规则 | 展示格式 | 备注 |
| 申请数 | 原子指标 | 计数（去重） | 来源表：credit_apply；字段：user_id | 当日去重用户数 | 整数 | |

Agent 行为：
  → 调用 save_knowledge(
      save_target="business",
      content="...",
      knowledge_type="requirement",
      title="授信全流程漏斗分析"
    )
  → 返回：已保存需求文档到业务文档知识库
```

### 开发 DWS 层 SQL

```
用户：请根据授信全流程漏斗需求，开发 DWS 层加工 SQL

Agent 行为：
  → 调用 search_knowledge(query="授信", search_scope="business")
    获取需求文档和业务规则
  → 调用 search_knowledge(query="授信申请", search_scope="schema", layer_filter="DWD")
    获取 DWD 层表结构
  → 调用 search_knowledge(query="授信结果", search_scope="schema", layer_filter="MID")
    获取 MID 层表结构
  → 结合知识库结果，按 Hive SQL 规范生成 DWS 加工 SQL
  → 执行检查清单验证
```
