# 常见错误检查清单

开发完成后逐项检查：

1. **表名缺少 schema 前缀**：所有表名必须带 `zijie.` 前缀
2. **LEFT JOIN 空值未处理**：SUM 聚合 LEFT JOIN 后的 flag 字段时，必须用 `COALESCE(flag, 0)` 包裹
3. **分区条件遗漏**：每个 CTE 或子查询中引用的表都必须加 `ds = '${bizdate}'`
4. **除零保护缺失**：所有除法运算分母必须用 `NULLIF(denominator, 0)` 保护
5. **字段名不一致**：SQL 中字段名必须与目标表 DDL 完全一致
6. **指标口径偏差**：计算逻辑必须与需求文档中的指标定义完全一致，不可自行推导或简化
7. **关联条件错误**：LEFT JOIN 的 ON 条件必须与需求文档中的"关联键"定义一致
8. **CTE flag 打标逻辑错误**：CASE WHEN 条件必须与需求文档中的"CTE flag 条件"完全一致
9. **派生指标计算位置错误**：派生指标（比率类）只能在 ADS 层计算，不能存入 DWS 层
10. **周/月表聚合方式错误**：可加指标用 SUM 聚合，不可加指标必须回源 DWD

## 注意事项

- 开发前先了解已有表结构，避免重复建设
- 上游层缺少所需字段时，提示用户先补充上游表
- 复杂业务逻辑先用文字描述确认后再编写 SQL
- 周表默认按自然周（周一~周日），特殊要求需与用户确认
- 所有回答和注释使用中文
- 需求文档中的"待确认项"在 SQL 注释中标注处理方式并提醒用户确认
- 需求文档表结构与现有表结构文档不一致时，必须先提示用户
